Overview 

Originally the ATI Accessibility Requirements Project Team created the CSU Accessibility Requirements to provide a clear standardized set of requirements that campuses can use to evaluate website accessibility and remediate accessibility errors. To provide campuses with a more tangible set of requirements the team has combined the current Section 508 guidelines, the WCAG 2.0 guidelines and the CSU Manual Evaluation  Guidelines into Compliance Sheriff tool. These criteria are designed and developed to capture accessibility issues and provide a detailed report. Each checkpoint group includes multiple checkpoints that target different accessibility issues.
